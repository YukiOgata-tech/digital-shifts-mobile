# Web → Native Staff Migration Progress

最終更新: 2026-07-24

## 完了条件

Web版でスタッフが実行できる日常業務を、管理者画面やPWA導線に依存せずiOS / Androidで完結できることを完了条件とする。画面の存在ではなく、実データの取得・入力・保存・エラー処理まで確認して判定する。

| スタッフ業務 | Web版の正本 | Native | 不足 |
|---|---|---|---|
| ログイン・セッション・ログアウト | 認証画面 | 完了 | パスワード再設定 |
| ホーム | `/app` | 一部 | Web版との表示項目差分確認 |
| 希望シフト提出 | `/app/shifts` | 一部 | 定義済み時間帯、全所属店舗の扱い |
| 自分の確定シフト | `/app/shifts` | 一部 | 月移動、過去シフト |
| 店舗の公開シフト表 | `/app/stores/[id]/shifts` | 未完 | 全スタッフ月間表、画像保存・共有 |
| 公開後のシフト調整申請 | shift adjustment panel | 未完 | 変更不可/時間変更/勤務可能/メモ、提出 |
| 打刻 | `/app/attendance` | 一部 | 実機のGPS・日跨ぎ確認 |
| 勤怠履歴・修正 | `/app/attendance/records` | 未完 | 履歴一覧、既存レコード修正 |
| 通知 | `/app/notifications` | 一部 | 通知リンクから対象画面への遷移 |
| ヘルプ募集 | help requests | 一部 | 応募メッセージ、応募履歴・取消可否 |
| プロフィール・設定 | settings/profile | 一部 | メール、パスワード、表示設定、アバター |
| Push通知 | notification service | 一部 | APNs/FCM、サーバー送信処理、実機確認 |

## 対象外

管理者向けシフト作成、給与計算、請求、developerコンソール、PWAインストール導線はスタッフ版完成後に扱う。

## 実装順

1. 店舗公開シフト表・月移動・画像保存/共有
2. 公開後のシフト調整申請
3. 勤怠履歴と既存レコード修正
4. 通知遷移、ヘルプ、設定の残差
5. 実データ・実機・権限拒否・アクセシビリティ検証
