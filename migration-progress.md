# Web → Native Staff Migration Progress

最終更新: 2026-07-25

## 完了条件

Web版でスタッフが実行できる日常業務を、管理者画面やPWA導線に依存せずiOS / Androidで完結できることを完了条件とする。画面の存在ではなく、実データの取得・入力・保存・エラー処理まで確認して判定する。

| スタッフ業務 | Web版の正本 | Native | 不足 |
|---|---|---|---|
| ログイン・セッション・ログアウト | 認証画面 | 完了 | 実機でメールリンク復帰を確認 |
| ホーム | `/app` | 一部 | Web版との表示項目差分確認 |
| 希望シフト提出 | `/app/stores/[id]/shift-periods/[periodId]` | 一部 | UI/操作は再現済み。共有DBへの原子的保存RPC/RLS適用、iOS/Android実機検証 |
| 自分の確定シフト | `/app/shifts` | 一部 | Web版と同じ5セクション構成、複数店舗横断、月間カレンダー、今後の勤務を実装。実機表示確認が必要 |
| 店舗の公開シフト表 | `/app/stores/[id]/shifts` | 一部 | 公開期間一覧、全スタッフ月間表、画像保存・共有を実装。大人数データの画像確認が必要 |
| 公開後のシフト調整申請 | shift adjustment panel | 一部 | Native画面と実データ操作を実装。RLS/RPC、通知、管理者反映後の結合確認 |
| 打刻 | `/app/attendance` | 一部 | 複数店舗一覧・詳細・例外理由まで実装。QRスキャナー、実機GPS・日跨ぎ確認が残る |
| 勤怠履歴・修正 | `/app/attendance/records` | 一部 | 月別履歴・店舗絞り込み・後から入力を実装。既存レコード修正が残る |
| 通知 | `/app/notifications` | 一部 | 通知リンクから対象画面への遷移 |
| ヘルプ募集 | help requests | 一部 | 応募メッセージ、応募履歴・取消可否 |
| プロフィール・設定 | `/app/settings`配下 | 一部 | 設定ハブ、プロフィール、メール変更、パスワード再設定、通知、曜日別初期値、端末データ方針を実装。英語UI全体、アバター、通知設定RPC migration適用が残る |
| Push通知 | notification service | 一部 | APNs/FCM、サーバー送信処理、実機確認 |

## 対象外

管理者向けシフト作成、給与計算、請求、developerコンソール、PWAインストール導線はスタッフ版完成後に扱う。

## 実装順

1. 希望シフトmigration適用とiOS/Android実機ジェスチャー検証
2. 店舗公開シフト表・月移動・画像保存/共有
3. 公開後のシフト調整申請
4. 勤怠履歴と既存レコード修正
5. 通知遷移、ヘルプ、設定の残差
6. 実データ・実機・権限拒否・アクセシビリティ検証
