# Dミセ ネイティブアプリ実装方針

## 目的

既存WebアプリのPWA導線を先に作るのではなく、スタッフが日常的に使う機能をiOS / AndroidのネイティブUIで完結させる。

## 初回リリース対象

1. 認証とセッション維持
2. 所属テナント・店舗の選択
3. スタッフホーム
4. 希望シフトの入力・下書き・提出・再編集
5. 自分と所属店舗の公開済みシフト確認、月移動、画像保存・共有
6. 公開後のシフト調整申請
7. 出勤・休憩・退勤打刻
8. GPSを利用した打刻条件の確認
9. 勤怠履歴・新規/既存勤怠の修正申請
10. 通知一覧・既読管理・対象画面への遷移
11. ヘルプ募集の確認・メッセージ付き応募・応募履歴
12. プロフィール・メール/パスワード・通知/表示設定・ログアウト

管理者向けシフト作成、給与計算、請求、開発者機能、PWAインストール導線は初回リリースの対象外とする。

## UI / UX方針

- Expo RouterのNative TabsとNative Stackを画面遷移の基盤にする。
- Web版のレイアウトを縮小移植せず、片手操作・大きなタップ領域・OS標準の戻る操作を優先する。
- 日付、選択、シート、ボタンは可能な範囲で`@expo/ui`のSwiftUI / Jetpack Compose実装を使う。
- 打刻や提出などの確定操作には`expo-haptics`で触覚フィードバックを付ける。
- 長い一覧はReact Nativeの仮想化リストを使い、設定系の短い一覧にはネイティブ形式を使う。
- 位置情報・通知などの権限は、その機能を初めて使う直前に説明して要求する。
- ライト・ダークモード、文字サイズ、VoiceOver / TalkBackを考慮する。

## ナビゲーション

- ホーム: 今日の勤務、提出期限、直近シフト、重要なお知らせ
- シフト: 希望シフト、自分の確定シフト、店舗の公開シフト表
- 打刻: 現在の勤務状態、出勤・休憩・退勤、当日の履歴
- その他: 通知、ヘルプ募集、勤怠修正、プロフィール・設定

## データ接続方針

- Supabaseの公開URLとPublishable Keyだけをアプリへ設定する。
- `service_role`やSecret Keyはモバイルアプリへ入れない。
- セッションはネイティブではSecureStoreへ分割保存し、URLセッション検出は無効化する。
- 読み書きは既存RLSを前提にし、権限昇格が必要な処理は既存API / RPC / Edge Functionへ寄せる。
- Web版のServer Actions、Cookie、相対URL APIはそのまま移植しない。
- 業務画面にデモデータを混在させず、設定不足・所属なし・取得失敗を明示する。
