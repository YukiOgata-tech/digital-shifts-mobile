## Supabaseへ追加済み

`D-mise`プロジェクトへ次のマイグレーションを適用済みです。

- `mobile_push_tokens`: 本人のみが自分の端末トークンを管理できるRLS付きテーブル
- `record_mobile_attendance_event`: 打刻レコードとイベントを1トランザクションで更新する認証必須RPC
- `20260724192044_align_mobile_attendance_with_web.sql`: 店舗位置未設定時の証跡判定をWEB版と統一
- `submit_mobile_manual_attendance`: 重複検証・イベント・監査ログをまとめる認証必須RPC
- anonロールから上記RPCの実行権限を明示的に剥奪

ローカルの正本は`supabase/migrations/`にあります。共有DBを利用するWeb版リポジトリにも同じマイグレーション履歴と生成型を同期する必要があります。

## Supabase適用待ち

`20260724162247_mobile_shift_request_workflow.sql`はローカルに作成済みですが、共有DBへの適用は未実施です。希望シフト全体の原子的置換、サーバー時刻による提出、提出済み・期限後・休業日・所属解除の拒否、`authenticated`限定実行、提出済み行の直接編集防止を追加します。既存Web版にも影響するRLS変更のため、対象プロジェクトを明示した承認後に適用し、Web版のmigration履歴へ同期します。

`202607250001_secure_mobile_notification_preferences.sql`も適用待ちです。スタッフ本人の通知設定更新を認証必須RPCへ集約し、メール通知を有効にする際は契約状態、プラン機能、期限内のfeature overrideをDB側で判定します。

## 既知の制約

- Push送信処理自体は、Web版の通知作成処理またはSupabase Edge FunctionからExpo Push Serviceを呼ぶ統合が必要です。秘密値はモバイルへ入れません。
- メール送信は既存Web版のサーバー責務です。Resend/SMTPの秘密値は`supabase/.env.example`に例示し、Expo公開envから分離しています。
- ブラウザCookieは使用しません。認証セッションはSecureStore、曜日別初期値・日付タップ動作・選択店舗は端末内、業務データはSupabaseに保存します。
- 表示言語は日本語を先行しています。Web版スタッフ画面にある英語切替は、モバイル全画面の翻訳完了後に有効化します。
- 管理者向けシフト作成、給与、請求、developerコンソール、PWA導線はスタッフ向けネイティブ初回リリースの対象外です。
- 希望シフトのなぞり入力はネイティブGesture Handlerで実装済みですが、指が画面外へ出る操作、OS中断、複数週の連続入力、VoiceOver / TalkBackは実機確認が必要です。
- Web版と現行DBは`shift_request_entries`を1日1件として扱うため、同じ日に複数の希望時間帯を登録するUI・保存形式は未対応です。要件化する場合はWeb/Native双方と集計処理のデータモデル変更が必要です。
- シミュレータは今回起動していません。実データでの端末操作確認は、テストアカウントと実機/EAS設定後に行います。
